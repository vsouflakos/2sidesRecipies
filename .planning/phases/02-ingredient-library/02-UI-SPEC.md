## Color

All values are the warm-minimal OKLCH tokens from Phase 1 `app.css`. Referenced by CSS variable name (light / dark handled automatically).

| Role | CSS Token (light) | CSS Token (dark) | Usage |
|------|-------------------|-----------------|-------|
| Dominant (60%) | `--background` oklch(0.981 0.007 60) | oklch(0.148 0.008 50) | Page background, empty search canvas |
| Secondary (30%) | `--secondary` / `--muted` oklch(0.948 0.010 60) | oklch(0.225 0.010 52) | Filter bar background, sidebar, detail page section backgrounds, table row hover |
| Accent (10%) | `--accent` oklch(0.550 0.035 60) | oklch(0.550 0.035 60) | See reserved list below |
| Destructive | `--destructive` oklch(0.577 0.245 27.325) | oklch(0.396 0.141 25.723) | Delete private ingredient button, destructive dialog confirm |

**Accent reserved for (exhaustive list — no other uses):**
1. Verified badge background (`bg-accent text-accent-foreground`)
2. Active filter pill / selected filter indicator
3. Search input focus ring (already handled by `--ring` token)
4. Primary CTA button background when using `variant="default"` (which maps to `--primary`, not accent — see note)

> Note: The warm-minimal primary (`--primary`) is near-black in light mode / near-white in dark mode. Primary buttons use `--primary`. Accent (the warm mid-tone) is strictly for trust signals (verified badge) and selected-state indicators. CTAs use `variant="default"` which maps to primary.

**Semantic color for "may contain" allergen state:** use `--muted-foreground` text on a `--muted` background chip — no new color needed. "Contains" allergen uses `--destructive` foreground at reduced opacity (text only, not background fill).

## Component Inventory

### Search & Browse Page (`ingredients/index.tsx`)

**Focal point:** The live search input is the primary visual anchor — full-width, prominent, positioned above all results and filters. Every other element on the page is subordinate to it; the page reads as a search tool first.

| Component | shadcn primitive | Notes |
|-----------|-----------------|-------|
| Search input | `Input` | Leading `SearchIcon` (lucide), 300ms debounce |
| Filter: Source | `Select` | Options: All / Official / My Ingredients |
| Filter: Allergen-free | `Popover` + `Checkbox` per allergen | 14 allergens from Phase 1 lookup |
| Filter: Verified only | `Checkbox` + `Label` | Inline with filter bar |
| Result rows | `div` (not `Table`) | Compact list; see row spec below |
| Allergen icons | `Tooltip` wrapping lucide icon or text badge | 14 EU allergens, icon + tooltip label |
| Verified badge | `Badge` variant="default" with accent bg | Text: "Verified" (EN) / "Επιβεβαιωμένο" (EL) |
| Skeleton rows | `Skeleton` | 8 skeleton rows shown while debounced reload is in flight |
| Empty state | Custom `div` | See copywriting section |
| Pagination | `Pagination` | 30 results per page (matches admin users pattern) |

**Search result row spec (compact density):**
- Height: 48px minimum
- Left: ingredient name — body (16px/400) in user's locale; secondary name in the other locale at label (14px/400) muted
- Middle: calories — label (14px/400) muted-foreground, format: `{N} kcal`
- Right: allergen icons (xs gap between icons, max 14 icons)
- Rightmost: verified badge (when applicable)
- Row divider: 1px `border-border`
- Row hover: `bg-muted` transition

## Interaction Contracts

### Live Search

- Debounce: 300ms (matches existing `UserController` pattern in admin users)
- Trigger: `router.reload({ only: ['ingredients'], preserveState: true, replace: true })`
- While loading: replace result rows with 8 `Skeleton` rows (48px height each)
- Min query length: 1 character (no minimum — clear query returns full paginated list)
- URL sync: query params `search`, `source`, `allergen_free[]`, `verified_only` always mirror applied filters

### Filter interactions

- Source filter (`Select`): immediate reload on change (no debounce needed — not typed input)
- Allergen-free filter (`Popover` + checkboxes): reload triggered on `Popover` close (not on each checkbox tick, to avoid multiple reloads)
- Verified only (`Checkbox`): immediate reload on change

### Duplicate-from-official starting point

- When "Duplicate from official" is selected, a `Command` combobox appears below the toggle
- Selecting an official ingredient pre-fills all form fields; a `(sonner)` toast confirms: "Pre-filled from [Ingredient Name]"
- The new private ingredient has no link back to the source (no FK to source — just a copy of data)
- Fields are editable after pre-fill

### Verify action (Mod/Admin)

- Button label: "Mark as Verified" when unverified; "Verified — Revoke" is NOT available (one-way; re-import can reset)
- On confirm: inline `Dialog` with body: "Confirm that all stored data for this ingredient is correct." + "Confirm Verification" and "Keep Unverified" buttons
- On success: badge appears, button disappears (permission gate remains but verified state is now true), sonner toast: "Ingredient verified."

### Price recording

- Form always visible above price history table
- After successful save: sonner toast "Price recorded." Form resets. History table updates inline (Inertia partial reload `only: ['prices']`).

### Private ingredient deletion

- `Dialog` confirmation: "Delete [Ingredient Name]? This cannot be undone." with "Delete" (destructive) and "Keep Ingredient" buttons
- On confirm: redirect to ingredient library index, sonner toast: "Ingredient deleted."

## Responsive Contract

All pages must be usable on desktop, tablet, and mobile (REQUIREMENTS UI-03 carried from Phase 1).

| Breakpoint | Search page | Detail page | Create form |
|------------|-------------|-------------|-------------|
| Mobile (<768px) | Full-width search input; filters collapse into a "Filters" `Sheet`; single-column result rows | Single-column tabs stack vertically | Single-column form; collapsible sections |
| Tablet (768–1024px) | Inline filter bar; result rows remain compact | Two-column nutrition grid | Two-column form fields where appropriate |
| Desktop (>1024px) | Filter bar inline; list takes 70% width; sidebar reserved for future detail preview | Three-column nutrition grid; Tab content in main column | Two-column form fields |

---

## Accessibility Contract

- All interactive elements meet 44px minimum touch target (WCAG 2.5.8)
- All color use meets WCAG AA contrast (warm-minimal tokens were verified in Phase 1 styleguide)
- Allergen icons must have `aria-label` with the allergen name (not icon-only without label)
- Search input must have visible `Label` or `aria-label="Search ingredients"`
- Filter `Checkbox` components must use shadcn `Checkbox` + `Label` association (already pattern in Phase 1)
- `Dialog` components use shadcn's built-in `DialogTitle` + `DialogDescription` (screen reader headings)
- Skeleton loading state must have `aria-busy="true"` on the list container
