**PerGramCostCalculator** (`app/Support/Ingredients/PerGramCostCalculator.php`) — namespace `App\Support\Ingredients`. A plain class with one public method:
```php
/**
 * Compute the per-gram cost of an ingredient price.
 * amount: money paid. quantity: how much was bought. unit: the unit of `quantity`.
 * Returns a BigDecimal per-gram cost, or throws/returns null when grams cannot be resolved.
 */
public function perGramCost(BigDecimal $amount, BigDecimal $quantity, Unit $unit, Ingredient $ingredient): ?BigDecimal
```
Logic:
- Resolve the total grams represented by `quantity` of `unit`:
  - If `$unit->type === 'weight'`: `grams = quantity * unit->base_factor` (base_factor is grams-per-unit; g=1, kg=1000). Use `BigDecimal::of($unit->base_factor)`.
  - If `$unit->type` is 'volume' or 'count': look up an `ingredient_conversions` row for this ingredient and this `from_unit_id`. If found, `grams = quantity * (conversion.gram_weight / conversion.from_amount)`. If no conversion exists, return `null` (the caller treats this as a validation failure — the ingredient has no conversion for that unit).
- `perGram = amount / grams` computed with `BigDecimal`, divided to 8 decimal places with `RoundingMode::HALF_UP`.
- All arithmetic via `Brick\Math\BigDecimal` — no PHP floats anywhere (RESEARCH Pitfall 4).

**StorePriceRequest** (`php artisan make:request Ingredients/StorePriceRequest`):
- `authorize()` returns `true` (any authenticated user may record a price on any ingredient they can see — pricing is per-user and private).
- `rules()`:
  - `amount` => `['required', 'numeric', 'gt:0']`
  - `quantity` => `['required', 'numeric', 'gt:0']`
  - `unit_id` => `['required', 'exists:units,id']`
  - `currency` => `['required', 'string', 'size:3']` (default EUR; other currencies allowed)
  - `recorded_at` => `['required', 'date', 'before_or_equal:today']`
  - `notes` => `['nullable', 'string', 'max:500']`
- Add a `withValidator()` (or an `after` validation hook) that, when the selected unit is not a weight unit, checks the ingredient has an `ingredient_conversions` row for that unit; if not, add an error on `unit_id` with message `__('app.ingredients.price_no_conversion')` (add this string to lang files if not present). Resolve the ingredient via the route param.

**IngredientPriceController** (`php artisan make:controller Ingredients/IngredientPriceController`) — single `store(StorePriceRequest $request, Ingredient $ingredient): RedirectResponse`:
