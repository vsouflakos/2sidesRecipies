**IngredientValidationRules trait** (`app/Concerns/IngredientValidationRules.php`) — a trait providing a `ingredientRules(): array` method with the shared rules:
- `name_en` => `['required_without:name_el', 'nullable', 'string', 'max:500']`
- `name_el` => `['nullable', 'string', 'max:500']`
- `category_id` => `['required', 'exists:ingredient_categories,id']`
- nutrition fields — for each of the 29 nutrition columns: `['nullable', 'numeric', 'min:0']`
- `allergens` => `['nullable', 'array']`; `allergens.*.allergen_id` => `['required_with:allergens', 'exists:allergens,id']`; `allergens.*.state` => `['required_with:allergens', 'in:contains,may_contain']`
- `conversions` => `['nullable', 'array']`; `conversions.*.from_amount` => `['required_with:conversions', 'numeric', 'gt:0']`; `conversions.*.from_unit_id` => `['required_with:conversions', 'exists:units,id']`; `conversions.*.gram_weight` => `['required_with:conversions', 'numeric', 'gt:0']`; `conversions.*.modifier` => `['nullable', 'string', 'max:100']`
Document the rule that the minimum required to save is name (at least one language) + category.

**StoreIngredientRequest** / **UpdateIngredientRequest** — `php artisan make:request Ingredients/StoreIngredientRequest` and the update one. Both `use IngredientValidationRules`, return `$this->ingredientRules()` from `rules()`. `authorize()`: Store returns `true` (any authenticated user may create a private ingredient). Update returns `$this->user()->can('update', $this->route('ingredient'))` so it routes through the policy.

**IngredientPolicy** (`php artisan make:policy IngredientPolicy --model=Ingredient`):
- `update(User $user, Ingredient $ingredient): bool` => `$ingredient->user_id !== null && $ingredient->user_id === $user->id` (owner only; official ingredients — user_id null — are never updatable here).
- `delete(User $user, Ingredient $ingredient): bool` => same condition.
Register it: in `AppServiceProvider::boot()` add `Gate::policy(Ingredient::class, IngredientPolicy::class);` (or rely on Laravel's auto-discovery if the project uses it — check AppServiceProvider; if no explicit policy registration exists, add the `Gate::policy` line and import `Illuminate\Support\Facades\Gate`).

**PrivateIngredientController** (`php artisan make:controller Ingredients/PrivateIngredientController`):
- `create(Request $request): Response` — renders `ingredients/create` with `categories` (the full 2-level tree, eager-loaded children), `allergens` (14), `units` (Phase 1 units). If a `?duplicate={id}` query param is present, load that ingredient (must be visible to the user — official or own private) with translations/allergens/conversions and pass it as a `duplicate` prop for pre-fill.
- `store(StoreIngredientRequest $request): RedirectResponse` — inside `DB::transaction()`: create the Ingredient with `user_id => auth()->id()`, `source => 'user'`, `source_id => (string) Str::uuid()`, `category_id`, `name_cache => $validated['name_en'] ?? $validated['name_el']`, and any provided nutrition columns. Create translations for each provided locale name. If `allergens` provided, build the pivot and `sync()`. If `conversions` provided, create conversion rows with `source => 'user'`. Redirect to the ingredient's detail route (or `ingredients.index` if 02-05 has not run yet — use `ingredients.index` as a safe target; 02-05 will not change this) with `->with('success', __('app.ingredients.toast_created'))`.
- `edit(Ingredient $ingredient): Response` — `$this->authorize('update', $ingredient);` then render `ingredients/create` with the same props plus `ingredient` (the existing record with relations) and `isEdit => true`.
- `update(UpdateIngredientRequest $request, Ingredient $ingredient): RedirectResponse` — authorization already enforced by the request; inside `DB::transaction()` update columns, re-sync translations / allergens / conversions. Redirect back with `__('app.ingredients.toast_updated')`.
- `destroy(Ingredient $ingredient): RedirectResponse` — `$this->authorize('delete', $ingredient);` then `$ingredient->delete()` (soft delete). Redirect to `ingredients.index` with `__('app.ingredients.toast_deleted')`.

**Routes** — in `routes/web.php`, inside the same authenticated group as `ingredients.index`, add (place `create` BEFORE any `{ingredient}` wildcard so it is not captured as an id):
```php
Route::get('ingredients/create', [PrivateIngredientController::class, 'create'])->name('ingredients.create');
Route::post('ingredients', [PrivateIngredientController::class, 'store'])->name('ingredients.store');
Route::get('ingredients/{ingredient}/edit', [PrivateIngredientController::class, 'edit'])->name('ingredients.edit');
Route::put('ingredients/{ingredient}', [PrivateIngredientController::class, 'update'])->name('ingredients.update');
Route::delete('ingredients/{ingredient}', [PrivateIngredientController::class, 'destroy'])->name('ingredients.destroy');
```
Add the `use App\Http\Controllers\Ingredients\PrivateIngredientController;` import. Run `vendor/bin/pint --dirty --format agent` and `npm run build`.
