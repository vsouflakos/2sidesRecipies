Backend routes available (from plans 05-02 + 05-03):
```
GET   recipes/{recipe}/conversation                          recipes.conversation.show     -> JSON {messages:[...]}
POST  recipes/{recipe}/conversation/stream                   recipes.conversation.stream   -> SSE: event:token data:{text}, event:done, event:error
POST  recipes/{recipe}/conversation/messages/{message}/apply  recipes.conversation.apply    -> JSON {status:'applied'|'failed', message?}
POST  recipes/{recipe}/conversation/messages/{message}/variant recipes.conversation.variant -> JSON {status, variant_recipe_id, variant_url}
```

Conversation message JSON shape (from recipes.conversation.show):
```typescript
type ConversationMessage = {
  id: number;
  role: 'user' | 'assistant' | 'tool_proposal';
  content: string;
  proposal_state: null | {
    kind: 'edit' | 'variant';
    action?: string;
    data?: unknown;
    changes?: unknown;
    status: 'pending' | 'applied' | 'dismissed' | 'failed';
    failure_reason?: string;
    variant_recipe_id?: number;
  };
};
```

SSE stream format (from RecipeConversationController::stream):
```
event: token\ndata: {"text":"..."}\n\n
event: done\ndata: {"finished":true}\n\n
event: error\ndata: {"message":"..."}\n\n
```

The builder uses `useRecipeAutosave` and `router` for draft updates. After an Apply succeeds,

i18n: translations live in lang/en/app.php and lang/el/app.php. Strings accessed in React via
