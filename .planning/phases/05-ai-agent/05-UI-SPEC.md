## Spacing Scale

Inherited from Phase 1 (established system — no changes). All values are multiples of 4, applied via Tailwind utilities.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding, message avatar gap |
| sm | 8px | Compact element spacing, message bubble internal padding (vertical), input area inner flex gap, starter prompt grid gap |
| md | 16px | Default card/sheet padding, message bubble horizontal padding, message bubble spacing, input area padding |
| lg | 24px | Sheet header padding, section spacing inside chat |
| xl | 32px | Chat panel header height area |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in chat sheet) |

**Exceptions:**

| Exception | Value | Reason |
|-----------|-------|--------|
| Chat send button (touch target) | min 44px height | WCAG 2.5.5 minimum touch target size |
| Starter prompt chips (touch target) | min 44px height | WCAG 2.5.5 — tappable on mobile |
| Apply / Dismiss buttons on proposal card | min 44px height | WCAG 2.5.5 — primary interaction in chat |

*Source: Phase 1 UI-SPEC (approved). Touch target exceptions per WCAG 2.5.5. All chat-surface spacing snaps to scale tokens — no off-scale values.*

## Color

Inherited from Phase 1 (approved token set in `resources/css/app.css`). No new CSS variables are introduced.

| Role | OKLCH (light) | Approx Hex | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `oklch(0.981 0.007 60)` | #FAF9F7 | Sheet background, page background behind sheet |
| Secondary (30%) | `oklch(0.948 0.010 60)` | #F2EFE9 | User message bubble background, muted sections |
| Accent (10%) | `oklch(0.550 0.035 60)` | #8C7A6A | See reserved-for list below |
| Destructive | `oklch(0.577 0.245 27.325)` | #DC2626 | Destructive actions only (not used in Phase 5 chat flow) |

**Accent (`--accent`) is reserved for Phase 5:**
1. The "Applied" badge on a proposal card that has been accepted and committed to the draft
2. The applied-edit marker (small badge or icon) on message bubbles that resulted in a draft change
3. The "Ask AI" trigger button's active/focus-visible ring when keyboard-navigated

Accent is NOT used for: the Send button (uses primary), agent message bubble background (uses card/muted), streaming indicator dots, error bubbles.

**Semantic colors for chat-specific states:**

| State | Color | OKLCH | Rationale |
|-------|-------|-------|-----------|
| Error bubble | `--destructive-foreground` text on `bg-destructive/10` | `oklch(0.577 0.245 27.325)` at 10% | Soft error surface — not a button, so full destructive fill would over-emphasize |
| Streaming/typing indicator | `--muted-foreground` | `oklch(0.530 0.018 55)` | Animated dots in secondary color |
| Agent message bubble | `--card` + `--border` border | `oklch(0.981 0.007 60)` | Distinct from page bg via border only — warm minimal, not heavy |
| User message bubble | `--secondary` | `oklch(0.948 0.010 60)` | Warm light grey — visually distinct from agent without high contrast |

*Source: Phase 1 UI-SPEC (approved). Phase 5 semantic color assignments are Claude's discretion within the established warm-minimal system.*

## Surface Contracts

### Surface 1: "Ask AI" Trigger (on `recipes/show.tsx`)

**Placement:** Recipe builder toolbar area, alongside "Save Version" and "Recall" buttons.

**Trigger states:**

| State | Visual |
|-------|--------|
| Provider configured + user is owner | Button: `variant="outline"` + `MessageCircle` Lucide icon + label "Ask AI" |
| Provider not configured | Button hidden entirely (not rendered) — `aiEnabled` prop gates rendering |
| Provider configured, user is NOT owner | Hidden (chat is owner-scoped) |

**Interaction:** Click opens the `AiChatSheet` (Sheet component, `side="right"`).

---

### Surface 2: AI Chat Sheet

**Component:** shadcn `Sheet` with `side="right"`. Mirrors the existing `VersionHistorySheet` pattern.

**Desktop (≥768px):**
- Width: 400px fixed
- Slides over the recipe builder (overlay, not push)
- Builder remains visible and interactive behind the sheet
- Close via X button in sheet header or clicking outside

**Mobile (<768px):**
- Full-screen overlay (100vw × 100dvh)
- Back arrow icon replaces X close button
- Same component, different `SheetContent` class via responsive Tailwind

**Sheet structure (top to bottom):**
1. SheetHeader: Title "Ask AI" (Heading, 20px/600) + close/back button
2. Separator
3. MessageList (scrollable, flex-1, auto-scroll to bottom)
4. Message input area (fixed to bottom of sheet): Textarea + Send button

**Empty state (no messages yet):** See StarterPrompts surface below.

**Loading state (conversation history loading):** 3 Skeleton rows in the message area — one wide (agent), one narrow-right (user), one wide (agent). Appears on sheet open before history loads via deferred Inertia prop.

---

### Surface 3: Message List

**Layout:** Vertically stacked message bubbles, newest at bottom, auto-scrolls on new message. `overflow-y-auto`, `flex flex-col gap-4` (16px between bubbles — `md` scale token).

**User message bubble:**
- Aligned right
- `bg-secondary` (warm light grey, `oklch(0.948 0.010 60)`)
- `rounded-lg` (0.5rem radius), `px-4 py-2` (16px/8px)
- Body text (16px/400, line-height 1.5)
- Max-width: 80% of sheet width

**Agent message bubble:**
- Aligned left
- `bg-card border border-border` (warm off-white with border)
- `rounded-lg`, `px-4 py-2`
- Body text (16px/400, line-height 1.5)
- Max-width: 90% of sheet width (agent messages are typically longer)
- If message contains a proposal card, the ProposalCard renders below the prose, inside the agent bubble container

**Applied-edit marker:**
- A small Badge (`variant="outline"`, accent-colored text) reading "Applied to draft" appears below the agent bubble that triggered an accepted edit
- Uses `--accent` color for badge text
- Positioned below and left-aligned with the bubble

**Role label (above each bubble group):**
- User: "You" — Label size (14px/400), muted-foreground, right-aligned
- Agent: "Chef AI" — Label size (14px/400), muted-foreground, left-aligned
- Label only appears on the first bubble of a consecutive group from the same role (not repeated for sequential messages)

---

### Surface 4: Streaming Indicator

**Appearance:** Three animated dots (`···`) inside an agent-styled bubble shell (same `bg-card border border-border` as agent messages).

**Animation:** Dots pulse with staggered opacity animation (0→1→0 with 300ms stagger per dot). Implemented via Tailwind `animate-pulse` or a custom CSS keyframe — no external animation library.

**Visibility:** Shown while fetch stream is in-flight. Hidden once `done` SSE event is received.

**Accessible:** `aria-label="Chef AI is typing"` on the container.

---

### Surface 5: Proposal Card

**Container:** shadcn `Card` component inside the agent message bubble area. `border border-border`, `rounded-md`, `p-4` (16px padding).

**Structure:**

```
┌─────────────────────────────────┐
│  [Edit icon]  Proposed Change   │  ← Heading (20px/600), "Pencil" Lucide icon
│                                 │
│  {Human-readable change summary}│  ← Body (16px/400), 1.5 line-height
│                                 │
│  [Apply]        [Dismiss]       │  ← Button row, right-aligned
└─────────────────────────────────┘
```

**Pending state (default):**
- "Apply" button: `variant="default"` (primary fill), min-height 44px
- "Dismiss" button: `variant="outline"`, min-height 44px
- Both buttons visible and interactive

**Applied state (after Apply is clicked):**
- Apply and Dismiss buttons replaced by a single Badge: `"Applied"` with accent color
- Card border changes to `border-accent/30` (soft accent tint)
- Card becomes non-interactive (pointer-events-none)
- Card stays visible in the message thread as a permanent record

**Dismissed state (after Dismiss is clicked):**
- Apply and Dismiss buttons replaced by a Badge: `"Dismissed"` with muted-foreground color
- Card opacity reduced to 60% (`opacity-60`)
- Card becomes non-interactive

**Failed state (Apply returned a validation error):**
- Apply and Dismiss buttons remain
- A red error text appears below the buttons (Label size, 14px, destructive color): `{error message}`
- The agent receives the failure as feedback automatically (server-side) — no user action needed

**Variant proposal card (AI-05):**
- Identical structure but heading reads "Create Recipe Variant"
- Applied state: Badge "Variant created" + a text link/button "Open variant →" (Body size, underline, primary color) replaces the badge — clicking navigates to the new recipe

### Surface 6: StarterPrompts (Empty State)

**When shown:** When the conversation has zero messages (first ever open of the chat for this recipe).

**Layout:** Centered vertically in the message area. `flex flex-col items-center gap-4`.

**Structure:**
1. `MessageCircle` Lucide icon, 32×32px, `text-muted-foreground`
2. Intro text (Body, 16px/400, muted-foreground, centered):
   "Ask me anything about this recipe, or try one of these:"
3. Grid of 4 prompt chips: `grid grid-cols-2 gap-2` (8px — `sm` scale token), full width of message area

**Prompt chip design:**
- shadcn `Button` with `variant="outline"`, `size="sm"`, `h-auto min-h-[44px]`, `text-left`, `whitespace-normal`
- Label size (14px/400), left-aligned text, wraps if needed
- Clicking a chip populates the Textarea with the prompt text and focuses it (does not auto-send)

**The 4 starter prompts (English source strings, translated at runtime via i18n):**

| # | English | Translation key |
|---|---------|-----------------|
| 1 | "Suggest a test for the current version" | `ai.starter.suggest_test` |
| 2 | "How can I lower the food cost?" | `ai.starter.lower_cost` |
| 3 | "Make a vegan variant" | `ai.starter.vegan_variant` |
| 4 | "What would improve the texture?" | `ai.starter.improve_texture` |

### Surface 7: Message Input Area

**Placement:** Fixed at the bottom of the Sheet, always visible. Visually separated from the message list by a top border (`border-t border-border`), `bg-background`, `p-4` (16px — `md` scale token) padding.

**Layout:** `flex gap-2 items-end` (8px — `sm` scale token)

**Textarea:**
- shadcn `Textarea`, `resize-none`, `min-h-[44px]`, `max-h-[120px]` (auto-expands up to 3 lines, scrolls beyond)
- Placeholder: "Ask Chef AI…" (muted-foreground)
- `onKeyDown`: Enter sends (without Shift), Shift+Enter inserts newline
- Disabled while a stream is in-flight

**Send button:**
- shadcn `Button`, `variant="default"` (primary fill), `size="icon"`, min 44×44px
- Icon: `Send` (Lucide), 16×16px
- Disabled while: textarea is empty OR stream is in-flight
- Loading state while stream is in-flight: shows `Spinner` icon instead of `Send`, button remains disabled
- `aria-label="Send message"`

---

### Surface 8: Error States

**Stream / API failure (turn-level error):**
- The in-flight streaming indicator is replaced by an error bubble (agent-aligned)
- Error bubble uses `bg-destructive/10 border border-destructive/20 text-destructive-foreground`
- Content: "Something went wrong. " + Retry button (inline, `variant="link"`, "Try again" label)
- Retry resends the last user message
- User's message is preserved in the Textarea for manual editing before retry

**Apply validation failure:**
- Error displayed inline inside the ProposalCard (see Surface 5 — failed state)
- No Sonner toast for validation errors (inline is clearer in context)

**Provider not configured:**
- "Ask AI" button is hidden entirely — no error state needed (Surface 1)

**Network error (fetch failed before any tokens):**
- Same error bubble as stream failure, same Retry treatment

### Surface 9: Responsive Behavior

| Breakpoint | Chat Sheet Behavior |
|------------|---------------------|
| Desktop (≥768px) | `Sheet side="right"`, 400px wide, overlays builder; builder visible and interactive behind it |
| Mobile (<768px) | `Sheet side="bottom"` with `h-[100dvh]` — full-screen; back arrow replaces X; builder not visible |

**Mobile-specific:**
- Starter prompts grid collapses to `grid-cols-1` (single column)
- Proposal card Apply/Dismiss buttons are full-width (`w-full`)
- The sheet uses `h-[100dvh]` to account for mobile browser chrome

## Accessibility Contract

- All interactive elements in the chat sheet meet WCAG 2.5.5 (min 44×44px touch target).
- StreamingIndicator: `aria-label="Chef AI is typing"` on the animated container.
- Send button: `aria-label="Send message"` (icon-only button requires label).
- Message list: `role="log"` with `aria-live="polite"` so screen readers announce new messages without interrupting the user.
- Proposal card Apply button: `aria-label="Apply proposed change: {summary}"` (dynamic, includes proposal summary for context).
- Sheet uses Radix Dialog's built-in focus trap — keyboard focus is contained within the sheet while open.
- Textarea `Enter` to send: documented in placeholder copy and does not conflict with accessibility because Shift+Enter preserves newline insertion.
- Starter prompt chips: `Button` elements (not `div`s) — keyboard focusable and activated via Enter/Space natively.
- Skeleton loading state: `aria-label="Loading conversation history"` on the skeleton container.
- Error bubble: `role="alert"` so screen readers announce the error immediately.
