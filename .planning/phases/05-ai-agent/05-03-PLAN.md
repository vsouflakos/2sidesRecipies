  <action>
    Create `app/Support/Recipes/SuggestionApplier.php` — the single point where an agent
    proposal becomes a draft edit, through the SAME validation path as a manual edit (AI-07).

    Constructor injects `RecipeDraftManager`.

    `public function apply(Recipe $recipe, RecipeConversationMessage $message): void`:
    1. Guard: if `$message->role !== 'tool_proposal'` or
       `data_get($message->proposal_state, 'status') !== 'pending'` — throw an
       `\InvalidArgumentException` (already-applied proposals cannot be re-applied).
    2. Guard: this method handles `kind === 'edit'` only. If `kind === 'variant'`, throw
       (variants are handled by the controller's `variant` action — Task 2).
    3. Ensure a draft exists: `$draft = $recipe->draft ?? RecipeDraft::create([...])` (mirror
       RecipeDraftController::update's default-draft creation).
    4. Resolve the new draft data: `$action = $message->proposal_state['action'];`
       `$newData = $message->proposal_state['data'] ?? [];`. If `$newData` is a field/value delta
       (`['field' => ..., 'value' => ...]`), apply it onto `$draft->data` with `data_set()` exactly
       as RecipeDraftController::update does; otherwise treat `$newData` as the full new data.
    5. VALIDATION (AI-07): run `$newData` through a Validator built from the SAME rules as
       `UpdateRecipeDraftRequest` — `Validator::make(['data' => $newData] + [...], (new UpdateRecipeDraftRequest)->rules())`.
       Use the `RecipeValidationRules` trait's `recipeDraftDataRules()` so the rule source is shared.
       If validation fails, set `proposal_state['status'] = 'failed'`,
       `proposal_state['failure_reason'] = <first validation message>`, save the message, and
       throw a `\Illuminate\Validation\ValidationException` (the controller maps it to a JSON error).
    6. CIRCULAR-REF GUARD: if `$action` is `add_sub_recipe`/`attach_sub_recipe`, run the same
       cycle check as RecipeDraftController (extract `wouldCreateCycleIncludingDrafts` into a
       shared `CircularReferenceDetector` method if not already there, or reuse the existing
       `App\Support\Recipes\CircularReferenceDetector`). On a cycle: mark the proposal 'failed'
       with reason "would create a circular reference" and throw.
    7. APPLY: call `$this->draftManager->applyEdit($draft, $action, $newData)` — exactly ONE call,
       so a bundled suggestion is one Recall step (CONTEXT.md "one Apply = one Recall step").
    8. On success: set `proposal_state['status'] = 'applied'`, save the message.
    Make the Wave 0 `applies an accepted suggestion` and `rejects an invalid suggestion` tests GREEN.
    Run `vendor/bin/pint --dirty --format agent`.
