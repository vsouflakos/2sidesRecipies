<behavior>
GramNormalizer.normalize(BigDecimal $quantity, Unit $unit, ?int $ingredientId): ?BigDecimal
  - weight unit → quantity * base_factor
  - volume/count unit with a matching IngredientConversion → quantity * (gram_weight/from_amount)
  - volume/count with no conversion, or null ingredientId → null (caller flags as a data gap)

NutritionCalculator.compute(array $lines, BigDecimal $portions, BigDecimal $totalYieldG): NutritionResult
  - line shape: {name, grams: BigDecimal, energy_kcal: ?string, protein_g: ?string, ...nutrient keys}
  - per nutrient: sum( value_per_100g * grams / 100 ) across all lines that have the value
  - perPortion = total / portions; per100g = total / totalYieldG * 100
  - lines missing energy_kcal collected into missingLines[]
  - METRIC-09: known fixture (e.g. 3 lines) yields an exact expected decimal string, no drift

CostCalculator.compute(array $lines, BigDecimal $portions, ?BigDecimal $sellingPricePerPortion): CostResult
  - line shape: {name, grams: BigDecimal, per_gram_cost: ?string}
  - totalCost = sum(per_gram_cost * grams); costPerPortion = totalCost / portions
  - foodCostPct = costPerPortion / sellingPricePerPortion * 100 (null when no selling price)
  - lines missing per_gram_cost collected into missingLines[]

ShrinkageCalculator.compute(array $lines): ShrinkageResult
  - line shape: {name, grams: BigDecimal, yield_pct: ?string}
  - rawTotal = sum(grams); cookedTotal = sum(grams * yield_pct/100, default 100 when null)
  - lossG = rawTotal - cookedTotal; lossPct = lossG / rawTotal * 100

BakersPercentageCalculator.compute(array $lines): BakersPercentageResult
  - line shape: {name, grams: BigDecimal, is_flour_base: bool, is_water: bool}
  - flourBase = sum(grams where is_flour_base) — supports multiple flour lines
  - each line percentage = grams / flourBase * 100
  - hydration = sum(grams where is_water) / flourBase * 100
  - when flourBase is zero → applicable=false, no division
</behavior>
