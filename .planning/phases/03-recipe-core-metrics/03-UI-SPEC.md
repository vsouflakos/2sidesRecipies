## Component Inventory

### Recipe List Page (`recipes/index.tsx`)

**Focal point:** The recipe grid is the primary visual canvas. Each card is a visual anchor communicating the recipe at a glance. The search input is always visible at the top; the filter panel is collapsible below it.

| Component | shadcn primitive | Notes |
|-----------|-----------------|-------|
| Search input | `Input` | Leading `SearchIcon` (lucide), 300ms debounce, same debounce pattern as ingredient index |
| Filter trigger | `Button` variant="outline" | "Filters" + active count badge; opens filter panel via `Sheet` on mobile, `Collapsible` on desktop |
| Filter: Tags | `Command` (multi-select combobox) | Free-form tags, existing tags shown as options |
| Filter: Cuisine | `Select` | Options from seeded cuisine list |
| Filter: Allergen | `Popover` + `Checkbox` per allergen | Same 14-allergen pattern as ingredient index; fires on Popover close |
| Filter: Ingredient | `Command` combobox | Search ingredient library to filter recipes containing that ingredient |
| Filter: Difficulty | `Select` | Easy / Medium / Hard / Expert |
| Filter: Time | `Select` | Ranges: Under 30 min / 30–60 min / 1–2 hours / Over 2 hours |
| Recipe card | `Card` | Grid layout; see card spec below |
| Skeleton cards | `Skeleton` | 6 skeleton cards shown while debounced reload is in flight |
| Empty state | Custom `div` | See copywriting section |
| Nav entry | Sidebar nav item | "Recipes" entry alongside existing "Ingredients" entry |

**Recipe card spec:**
- Width: fills grid column; grid is 1-column on mobile, 2-column on tablet, 3-column on desktop
- Hero image: 16:9 aspect ratio at top of card; if no hero image, show a neutral `bg-muted` placeholder with centered `UtensilsCrossedIcon` (lucide) at 32px, `text-muted-foreground`
- Recipe name: Display size (28px/600) on card hover expands to truncated 2-line clamp; default 1-line clamp at 20px/600 — use Heading size (20px) on the card to keep visual balance across the grid
- Cuisine badge: `Badge` variant="secondary", Label size (14px), positioned below name
- Metadata row: total time (prep + cook), difficulty badge — Label (14px/400), muted-foreground
- Metrics row: cost per portion (formatted `€N.NN / portion`) + calories per portion (`N kcal`) — Label (14px/400)
- Allergen icons: up to 14, xs gap, with `Tooltip` on hover — same pattern as ingredient rows
- Card hover: subtle `shadow-md` lift; recipe name underline

| State | Behaviour |
|-------|-----------|
| Empty (no query) | Shows placeholder: "Search ingredients or recipes…" |
| Typing | Debounced 300ms; calls `GET /search/components?q=` endpoint; returns flat list `{type, id, name, unit_hint}` max 10 items |

## Interaction Contracts

### Auto-save (working draft)

- Trigger: any change to ingredient lines, steps, section structure, metadata fields, chef notes
- Debounce: 600ms after the last keystroke / interaction
- Mechanism: `router.put(updateDraft(...), draftData, { preserveState: true, preserveScroll: true, only: ['draft', 'metrics'] })`
- Visual indicator: subtle "Saving…" / "Saved" label at Label size (14px/400) in `text-muted-foreground` next to the builder header — disappears 2s after "Saved"; implement with Tailwind utility `text-sm` (no new design-system size declared)
- No toast on auto-save (silent background operation)

### Save Version

- User clicks "Save Version" in metrics panel
- Optional: `Dialog` prompts for a short "What changed?" note (textarea, max 140 chars); "Save without note" secondary action skips the note
- On success: version history updates, "v{N}" badge in panel updates, sonner toast: "Version v{N} saved."
- On failure: sonner toast with error; draft preserved

### Recall

- User clicks "Recall" in metrics panel
- No confirmation dialog — Recall is immediately applied (it is reversible by re-doing the action)
- On success: builder reverts one logical edit; draft reloads via `only: ['draft', 'metrics']` partial; sonner toast: "Edit undone."
- "Recall" button disabled when edit log is empty (no undoable edits)
- Accepts `expected_sequence` in request body; on 409 (sequence mismatch): sonner toast "Draft updated elsewhere — please refresh" and disable Recall until refresh

### Scaling (view-only what-if)

- User changes scale factor or portion count in metrics panel scaling controls
- All ingredient line quantities and all metrics panel values recompute instantly in the React layer (no server round-trip for view-only scaling)
- Quantities shown in italic to signal "what-if, not saved" state
- "Apply to Draft" button appears when scale/portions differ from draft values
- Clicking "Apply to Draft": sends scaled quantities and new portions to the draft update endpoint; treated as one logical edit (one Recall step); sonner toast: "Scale applied to draft."

### Sub-recipe pin / update cue

- Sub-recipe line displays pinned version badge: `v{N}` in `Badge` variant="secondary"
- When a newer version of the component exists: accent-colored "Update available" `Badge` appears inline on the line row
- Clicking "Update available" badge opens a `Dialog`: "Update '{sub-recipe name}' from v{old} to v{new}? This will change the metrics for this recipe." with "Update Pin" and "Keep v{old}" buttons
- Updating pin is one logical edit (Recall-able)

### Quick-create ingredient (from builder)

- Triggered from "Create '{query}' as private ingredient" option in the combobox
- Opens `Dialog` (not a page navigation) with minimal form: name (EN) pre-filled from query, category `Select`, submit "Save and Add"
- On success: dialog closes; ingredient is added as a line immediately; full ingredient editing available later via ingredient library
- On failure: inline form validation errors within the dialog

### Recipe deletion

- Accessed via `DropdownMenu` on the recipe card or builder header
- `Dialog` confirmation: "Delete '{recipe name}'? All versions will be permanently removed." with "Delete Recipe" (destructive) and "Keep Recipe" cancel
- On confirm: redirect to recipe list; sonner toast: "Recipe deleted."

### Recipe duplication

- Accessed via `DropdownMenu` on recipe card or builder header
- No confirmation (non-destructive action)
- On success: redirect to new recipe's builder (new recipe named "{source name} (copy)"); sonner toast: "Recipe duplicated. You're now editing the copy."

### Circular reference rejection

- Detected server-side in `CircularReferenceDetector` before attaching the sub-recipe line
- Returned as a 422 validation error on the draft update endpoint
- Surfaced in the builder as an inline error below the combobox row: "Cannot add '{recipe name}' — it would create a circular reference. Recipe A → B → A is not allowed."
- No toast for this error (inline is sufficient and more contextual)

### Filter interactions (recipe list)

- Search input: 300ms debounce, `router.reload({ only: ['recipes'], preserveState: true, replace: true })`
- Cuisine, Difficulty, Time `Select` filters: immediate reload on change
- Allergen `Popover` + checkboxes: reload on Popover close (same contract as ingredient allergen filter)
- Tags `Command` combobox: reload on item select/deselect
- Ingredient `Command` combobox: reload on item select; "Remove" pill to clear
- Active filter count badge on "Filters" button: count of non-default filters currently applied

## Responsive Contract

All pages follow the Phase 1 UI-03 requirement: usable on desktop, tablet, and mobile.

| Breakpoint | Recipe list | Builder | Metrics panel |
|------------|-------------|---------|---------------|
| Mobile (<768px) | Single-column card grid; "Filters" opens `Sheet`; hero image 16:9 | Single column; metadata block collapsed; section blocks full-width | Collapses to a pinned summary bar at bottom of viewport showing cost/portion + kcal/portion; tap to expand full panel as a `Sheet` from bottom |
| Tablet (768–1024px) | Two-column card grid; filter panel collapses inline below search | Single column builder; metrics panel as a full-width block below the builder | Full metrics panel below builder columns |
| Desktop (>1024px) | Three-column card grid; filter panel as a collapsible sidebar left or inline bar | Two-column: builder 65% left, metrics panel 35% right, sticky | Sticky right column; scrolls independently |

## Accessibility Contract

- All interactive controls meet 44px minimum touch target (WCAG 2.5.8)
- All color use meets WCAG AA contrast (warm-minimal OKLCH tokens verified in Phase 1)
- Ingredient line combobox must have `aria-label="Search ingredients or recipes"`
- Allergen chips in metrics panel must include allergen name as visible text (not icon-only)
- `Dialog` components use `DialogTitle` + `DialogDescription` (shadcn built-in)
- Skeleton loading states have `aria-busy="true"` on the list/grid container
- Version diff highlights (bg-yellow) must not rely on color alone — changed rows also get a visual indicator icon (`DiffIcon` or `ArrowRightIcon` at row start)
- Recall button: `aria-disabled="true"` (not `disabled`) when edit log is empty, with tooltip: "Nothing to undo"
- Up/down reorder buttons must have `aria-label="Move section up"` / `aria-label="Move step down"` etc.
