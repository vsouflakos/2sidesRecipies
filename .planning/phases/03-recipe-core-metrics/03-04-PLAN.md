<objective>
Build the recipe backend: routes, RecipePolicy, FormRequests, the five recipe controllers, the unified component-search endpoint, the version/duplicate logic, and the `RecipeMetricsService` that composes Plan 02 calculators + Plan 03 aggregator into a single metrics prop for Inertia. This is the server contract every Phase 3 page consumes.

Purpose: The builder, metrics panel, sub-recipe, and list UIs all read Inertia props from these controllers and POST/PUT to these routes. Wiring the backend here lets all four UI plans run against a stable contract.
Output: Routes + policy + 3 FormRequests + 5 controllers + 2 services + 2 resources.
</objective>
