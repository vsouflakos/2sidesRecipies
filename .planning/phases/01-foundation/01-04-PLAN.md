    Create `app/Concerns/AdminUserGuards.php` — a trait with the shared guard logic:

    ```php
    <?php

    namespace App\Concerns;

    use App\Models\User;

    trait AdminUserGuards
    {
        protected function isLastAdmin(User $target): bool
        {
            if (! $target->hasRole('Admin')) {
                return false;
            }

            return User::role('Admin')->count() <= 1;
        }

        protected function isActingOnSelf(User $target): bool
        {
            return auth()->id() === $target->id;
        }
    }
    ```

    Create three FormRequests via `php artisan make:request Admin/AssignRoleRequest --no-interaction` (and the
    two others). Each `authorize()` returns `true` (route is already permission-gated). Each uses the
    `AdminUserGuards` trait and adds a `withValidator()` block.

    `app/Http/Requests/Admin/AssignRoleRequest.php`:
    - `rules()`: `['role' => ['required', 'string', \Illuminate\Validation\Rule::in(['User', 'Moderator', 'Admin'])]]`
    - `withValidator(Validator $validator)`: `$validator->after(function ($v) { ... })`:
      - `$target = $this->route('user');`
      - If `$this->isActingOnSelf($target)` → `$v->errors()->add('role', __('app.admin.guard_self_role'));`
      - If `$this->isLastAdmin($target) && $this->input('role') !== 'Admin'` → `$v->errors()->add('role', __('app.admin.guard_last_admin_role'));`

    `app/Http/Requests/Admin/DeactivateUserRequest.php`:
    - `rules()`: return `[]` (no body fields — status toggles based on current state).
    - `withValidator()` after-callback:
      - If `$this->isActingOnSelf($target)` → add error key `status` with `__('app.admin.guard_self_deactivate')`
      - If `$this->isLastAdmin($target)` → add error key `status` with `__('app.admin.guard_last_admin_deactivate')`

    `app/Http/Requests/Admin/DeleteUserRequest.php`:
    - `rules()`: return `[]`.
    - `withValidator()` after-callback:
      - If `$this->isActingOnSelf($target)` → add error key `user` with `__('app.admin.guard_self_delete')`
      - If `$this->isLastAdmin($target)` → add error key `user` with `__('app.admin.guard_last_admin_delete')`

    Create the controller via `php artisan make:controller Admin/UserController --no-interaction`.
    Implement four thin methods. Wrap role/status/delete mutations in `DB::transaction()` with a
    `lockForUpdate()` on the target user (RESEARCH Pitfall 5):

    - `index(Request $request): Response`
      - Read `$search = $request->string('search')->toString();`
      - Query: `User::query()->with('roles')->when($search, fn ($q) => $q->where(fn ($w) => $w->where('name', 'like', "%{$search}%")->orWhere('email', 'like', "%{$search}%")))->orderBy('name')->paginate(15)->withQueryString();`
      - Transform each user to include `id, name, email, role` (first role name), `account_status` (string value),
        `created_at` (ISO), `is_self` (`$u->id === auth()->id()`), `is_last_admin` (true when the user is an Admin and `User::role('Admin')->count() <= 1`).
      - `return Inertia::render('admin/users', ['users' => $paginated, 'filters' => ['search' => $search]]);`
    - `assignRole(AssignRoleRequest $request, User $user)`
      - `DB::transaction(function () use ($request, $user) { $user->syncRoles([$request->validated('role')]); });`
      - `return back()->with('success', __('app.admin.toast_role_changed', ['name' => $user->name, 'role' => $request->validated('role')]));`
    - `toggleStatus(DeactivateUserRequest $request, User $user)`
      - Toggle: if currently `AccountStatus::Active` set `Deactivated`, else set `Active`. Save inside a transaction.
      - On deactivation flash `__('app.admin.toast_deactivated', ['name' => $user->name])`; phrase the success message based on the new state.
    - `destroy(DeleteUserRequest $request, User $user)`
      - `DB::transaction(fn () => $user->delete());` (SoftDeletes — `delete()` soft-deletes).
      - `return back()->with('success', __('app.admin.toast_deleted', ['name' => $user->name]));`

    Add `use Illuminate\Support\Facades\DB;`, `use App\Enums\AccountStatus;`, and the FormRequest/`User`/`Inertia` imports.

    NOTE: the `/admin/users` routes already exist in `routes/web.php` (declared by Plan 02). Do NOT re-declare them.

    Run `vendor/bin/pint --dirty --format agent`.
