## Spacing Scale

Standard 8-point scale. All values are multiples of 4. Applied via Tailwind utilities (`p-1` = 4px, `p-2` = 8px, etc.) mapped through the Tailwind v4 `@theme` block already in `resources/css/app.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding |
| sm | 8px | Compact row padding, inline button padding |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Section internal padding, table cell padding |
| xl | 32px | Layout column gaps, page section breaks |
| 2xl | 48px | Page-level vertical rhythm between major sections |
| 3xl | 64px | Styleguide swatch grid gutters, full-page top padding |

**Exceptions:**

| Exception | Value | Reason |
|-----------|-------|--------|
| Touch targets (language switcher, inline role select, deactivate/delete buttons) | min 44px height | WCAG 2.5.5 minimum touch target size |
| Admin table row height | 52px | Comfortable density for data table with inline controls |

*Source: Default — standard 8-point scale. Touch target exception per WCAG. Table row height is Claude's discretion.*

## Typography

Single typeface throughout: **Instrument Sans** (already installed). Heading contrast is achieved via weight and size only — no second typeface. Greek characters (El locale) are fully supported by Instrument Sans.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (Semibold) | 1.2 | Page headings (e.g. "User Management", styleguide section titles) |
| Heading | 20px | 600 (Semibold) | 1.2 | Card headings, section sub-headings, dialog titles |
| Body | 16px | 400 (Regular) | 1.5 | Table cell text, form labels, paragraph copy |
| Label | 14px | 400 (Regular) | 1.4 | Table column headers, badge text, helper text, nav items |

**Only two weights used:** 400 (Regular) and 600 (Semibold). Bold (700) is not used in Phase 1.

*Source: CONTEXT.md ("single clean sans-serif, heading contrast via weight/size, not a different family"). Sizes and weights are Claude's discretion within that direction.*

### 60/30/10 Split

| Role | OKLCH (light) | Usage |
|------|---------------|-------|
| Dominant (60%) | `oklch(0.981 0.007 60)` — warm off-white | Page background, card surfaces, popover surfaces |
| Secondary (30%) | `oklch(0.948 0.010 60)` — warm light grey | Secondary button fills, muted sections, table header row, disabled row backgrounds, sidebar hover |
| Accent (10%) | `oklch(0.550 0.035 60)` — warm taupe | See reserved-for list below |
| Destructive | `oklch(0.577 0.245 27.325)` — standard red | Destructive actions only |

**Accent (`--accent`) is reserved for:**
1. The active/selected state indicator on the language switcher (EN / EL toggle — the selected locale tab gets the accent background)
2. Focus-visible ring on admin table action buttons (role select, deactivate, delete) when keyboard-navigated
3. Styleguide page: the accent color swatch label and its preview tile border

Accent is NOT used for: primary buttons, nav active states, links, badges, or any general interactive hover state.

*Source: CONTEXT.md ("warm taupe / near-neutral accent — very soft, low saturation, used sparingly"). Token values from RESEARCH.md Pattern 9 (OKLCH baseline). 60/30/10 split and reserved-for list are Claude's discretion.*

### Surface 2: `/admin` Shell + User List Page

**Route:** `GET /admin/users` → `resources/js/pages/admin/users.tsx`

**Layout:** Mirror the settings layout pattern (`resources/js/layouts/settings/`). Admin sidebar entry: label "Users" with a `Users` Lucide icon, visible only when the authenticated user has the `manage-users` permission.

**Page structure (top to bottom):**
1. Page heading: "User Management" (Display size, weight 600)
2. Search row: Input (placeholder: "Search by name or email…") aligned left, result count aligned right ("Showing N of M users")
3. Data table
4. Pagination controls

**Table columns:**

| Column | Content | Width | Notes |
|--------|---------|-------|-------|
| User | Avatar + Name + Email (stacked) | flex-1 | Name in Body weight 400, email in Label/muted-foreground |
| Role | Badge (variant matches role: Admin = primary, Moderator = secondary, User = outline) | 120px | |
| Joined | Formatted date (e.g. "16 May 2026") | 140px | Label size, muted-foreground |
| Actions | Dropdown menu trigger (ellipsis icon) | 48px | See action states below |

**Action states:**

| Action | Trigger | Normal state | Guarded state |
|--------|---------|-------------|---------------|
| Change role | Dropdown → "Change Role" → Select | Opens role Select inline | Disabled if row is self OR if row is last Admin; Tooltip: "You cannot change your own role" / "Cannot remove the last Admin" |
| Deactivate | Dropdown → "Deactivate" | Opens Dialog confirmation | Disabled if row is self OR if row is last Admin; Tooltip explains |
| Delete | Dropdown → "Delete account" | Opens Dialog confirmation | Disabled if row is self OR if row is last Admin; Tooltip explains |

**Disabled control appearance:** Reduced opacity (0.5), cursor-not-allowed. The dropdown menu item is rendered as disabled (shadcn DropdownMenuItem `disabled` prop), not hidden. Reason is surfaced via a Tooltip wrapping the disabled item.

**Empty state (no users match search):**
- Heading: "No users found"
- Body: "No users match your search. Try a different name or email."
- No CTA (search is already visible above the table)

**Loading state:** Skeleton rows — 5 rows of Avatar + two text blocks + Badge skeleton + date skeleton.

**Toast feedback (Sonner):**
- Role changed: "Role updated — [Name] is now a [Role]."
- User deactivated: "[Name]'s account has been deactivated."
- User deleted: "[Name]'s account has been deleted."
- Error: "Something went wrong. Please try again." (on server error)

### Surface 3: Deactivate Confirmation Dialog

- Loading state: Deactivate account button shows Spinner, is disabled.

### Surface 4: Delete Confirmation Dialog

**Trigger:** Dropdown "Delete account" on an eligible user row.

**Dialog content:**
- Title: "Delete account?"
- Body: "This will permanently delete [Name]'s account and all their data. This action cannot be undone."
- Actions: [Keep account] (secondary button) + [Delete account] (destructive button)
- Loading state: Delete account button shows Spinner, is disabled.

*Note: SoftDeletes means data is soft-deleted; the copy says "cannot be undone" from the user's perspective — recoverable only via database. Do not surface recovery in this dialog.*

### Surface 5: Language Switcher Component

**File:** `resources/js/components/language-switcher.tsx`

**Placement:** App sidebar bottom area (below nav items, above user/settings area). Mirrors the appearance-mode toggle placement pattern in the existing sidebar.

**Interaction:**
- Two Toggle buttons side by side: "EN" and "EL"
- Selected locale: accent background (`bg-accent`, `text-accent-foreground`), weight 600
- Unselected locale: transparent background, `text-muted-foreground`, weight 400
- Selecting a locale: optimistic UI update via `setLocale()` hook, then `router.put(route('locale.update'), { locale })` to persist. No confirmation needed.
- Both buttons have a minimum height of 44px for touch targets.

**Size:** Label (14px). Do not use a Select or Dropdown — the two-button toggle is the correct pattern for a binary language choice.

## Accessibility Contract

- All interactive elements meet WCAG 2.5.5 (minimum 44×44px touch target).
- Contrast: warm off-white background (`oklch(0.981 0.007 60)`) against foreground (`oklch(0.168 0.010 50)`) achieves ≥7:1 contrast ratio (AAA).
- Contrast: muted-foreground (`oklch(0.530 0.018 55)`) against background achieves ≥3:1 (AA for large text / UI components).
- Destructive (`oklch(0.577 0.245 27.325)`) on white background achieves ≥4.5:1 (AA).
- Focus rings use `--ring` token. All shadcn/ui components expose `focus-visible:ring-ring` by default.
- Disabled action dropdown items use `aria-disabled="true"` (shadcn DropdownMenuItem `disabled` prop handles this).
- Dialogs use Radix Dialog's built-in focus trap and `aria-modal`.
- Language switcher Toggle pair uses `role="group"` and `aria-label="Language"`.
- Admin table uses semantic `<table>` / `<thead>` / `<tbody>` / `<th scope="col">` via the shadcn Table component.
- Admin table actions column: the ellipsis/kebab menu trigger button must carry `aria-label="Actions for {user full name}"` (e.g. `aria-label="Actions for Maria Papadopoulou"`). This label is set dynamically per row using the user's full name so screen-reader users can distinguish triggers across rows.
