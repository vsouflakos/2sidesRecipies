RecipeController::index (the filter pattern to MIRROR for LibraryController::index):
  Reads request params: search (string), tag, cuisine, allergen (string),
  ingredient, difficulty (string), max_total_time (int).
  Query: Recipe::query()->where('user_id', auth()->id())
    ->with(['currentVersion','cuisine','tags'])
    ->when($search, fn($q)=>$q->where('name','like',"%{$search}%"))
    ->when($tag, fn($q)=>$q->whereHas('tags', fn($t)=>$t->where('tags.id',$tag)))
    ->when($cuisine, fn($q)=>$q->where('cuisine_id',$cuisine))
    ->when($allergen, fn($q)=>$q->whereHas('currentVersion',
        fn($v)=>$v->whereJsonContains('cached_allergen_slugs->contains',$allergen)))
    ->when($ingredient, ...)  // <- OMIT for library (excluded per UI-SPEC)
    ->when($difficulty, fn($q)=>$q->where('difficulty',$difficulty))
    ->when($maxTotalTime, fn($q)=>$q->whereRaw(
        '(COALESCE(prep_time_minutes,0)+COALESCE(cook_time_minutes,0)) <= ?',[$maxTotalTime]))
    ->orderByDesc('updated_at')
    ->paginate(24)->withQueryString()
    ->through(fn(Recipe $r)=>(new RecipeListResource($r))->resolve());
  Returns Inertia::render('recipes/index', [...]).

RecipeListResource::toArray output keys (the shape PublicRecipeListResource mirrors):
  id, name, slug, hero_image_path, cuisine, total_time, difficulty,
  cost_per_portion (OMIT in public), calories_per_portion, allergen_slugs.
  calories_per_portion is extracted from version cached_nutrition_json
  ['per_portion']['energy_kcal'] ?? ['totals']['energy_kcal'].
  allergen_slugs is flattened from version cached_allergen_slugs which is
  {contains: string[], may_contain: string[]} OR a flat slug array.

Recipe model (after Plan 01):
  Columns is_published (bool), published_version_id (nullable), published_at
  (datetime, indexed). Relations currentVersion(), publishedVersion()
  [belongsTo RecipeVersion via published_version_id], user(), cuisine(), tags().

RecipeVersion model — columns: version_number, committed_at, change_note,
  snapshot (array — draft JSON with sections[].lines[] each possibly carrying
  sub_recipe_version_id), cached_nutrition_json (array), cached_allergen_slugs
  (array), cached_cost_per_gram, cached_cost_per_portion, cached_selling_price.
  PUBLIC resources read cached_nutrition_json and cached_allergen_slugs only —
  NEVER cached_cost_* and NEVER re-resolve ingredient_id against the live
  ingredients table (RESEARCH Pattern 7 / Anti-Patterns).

RecipePolicy (after Plan 01): view(?User, Recipe), update(User, Recipe),
  delete(User, Recipe). update() is owner-only — publish/unpublish authorize
  with Gate::authorize('update', $recipe).

Route declaration convention: static segments declared BEFORE wildcard segments.
Existing route names in routes/web.php: recipes.index, recipes.create,
recipes.store, recipes.show ('recipes/{recipe}'), recipes.destroy. Controllers
are thin and feature-grouped (app/Http/Controllers/Recipes/, app/Http/Controllers/Admin/).
Existing pattern: RecipeDuplicateController, RecipeDraftController are
single-responsibility controllers — follow that for PublishRecipeController.
